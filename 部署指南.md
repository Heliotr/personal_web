# 部署指南：将网站部署到阿里云轻量服务器

## 整体流程

```
本地构建 → 上传代码 → 服务器安装依赖 → 启动服务 → 配置 Nginx → HTTPS
```

---

## 第一步：本地构建

在本地项目目录执行：

```bash
npm run build
```

构建成功后，会在 `.next` 目录生成生产文件。

---

## 第二步：上传代码到服务器

### 方法 A：Git 克隆（推荐）

1. 在服务器上安装 Git（如果未安装）：
   ```bash
   # Ubuntu/Debian
   apt update && apt install -y git

   # CentOS/Rocky/Alibaba Cloud Linux
   yum install -y git
   ```

2. 克隆仓库：
   ```bash
   cd /opt
   git clone https://github.com/Heliotr/personal_web.git
   cd personal_web
   ```

3. 安装依赖：
   ```bash
   npm install
   ```

### 方法 B：FTP/SFTP 上传

1. 使用 FileZilla 或 WinSCP 等工具
2. 连接到服务器 IP
3. 上传整个项目文件夹到 `/opt/personal_web`
4. SSH 登录服务器后执行 `npm install`

---

## 第三步：安装 Node.js 和 PM2

### 安装 Node.js（如果服务器没有）

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# CentOS/Rocky/Alibaba Cloud Linux
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs
```

### 安装 PM2

```bash
npm install -g pm2
```

---

## 第四步：启动服务

```bash
cd /opt/personal_web

# 使用 PM2 启动
pm2 start npm --name "personal_web" -- start

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

---

## 第五步：配置 Nginx

### 1. 安装 Nginx（如果未安装）

```bash
# Ubuntu/Debian
apt update && apt install -y nginx

# 启动并设置开机自启
systemctl enable nginx
systemctl start nginx

# CentOS/Rocky
yum install -y nginx

# 启动并设置开机自启
systemctl enable nginx
systemctl start nginx
```

### 2. 申请 SSL 证书（可选，已有可跳过）

使用 Let's Encrypt 免费证书：

```bash
# Ubuntu/Debian
apt update && apt install -y certbot python3-certbot-nginx

# CentOS/Rocky
yum install -y epel-release
yum install -y certbot python3-certbot-nginx

# 申请证书（需要域名已解析到服务器）
certbot --nginx -d cat111.cn -d www.cat111.cn
```

**注意**：如果申请失败（域名未解析/备案问题），可以先跳过 SSL，用 HTTP 测试。

### 3. 创建 Nginx 配置文件

```bash
vi /etc/nginx/conf.d/cat111.cn.conf
```

添加以下内容（已有 SSL 证书的用这个）：

```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name cat111.cn www.cat111.cn;
    return 301 https://$server_name$request_uri;
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name cat111.cn www.cat111.cn;

    # SSL 证书路径（根据你的实际路径修改）
    ssl_certificate /etc/letsencrypt/live/cat111.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/cat111.cn/privkey.pem;

    # SSL 配置
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

**如果没有 SSL 证书**，用这个简化配置（HTTP only）：

```nginx
server {
    listen 80;
    server_name cat111.cn www.cat111.cn;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4. 测试并重启 Nginx

```bash
# 测试配置
nginx -t

# 重启 Nginx
nginx -s reload
```

## 第六步：更新代码

以后更新网站时，在服务器上执行：

```bash
cd /opt/personal_web

# 拉取最新代码
git pull

# 重新构建
npm run build

# 重启 PM2
pm2 restart personal_web
```

## 常见问题

### 1. 端口被占用
```bash
# 查看 3000 端口占用
lsof -i:3000

# 杀掉占用进程
kill -9 <PID>
```

### 2. PM2 不自动启动
```bash
# 重新生成 startup 脚本
pm2 unstartup
pm2 startup
pm2 save
```

### 3. SSL 证书申请失败
- 域名需要解析到服务器 IP
- 确保防火墙开放 80 和 443 端口
- 阿里云安全组需要放行 80/443 端口

### 4. 网站无法访问
- 检查 PM2 状态：`pm2 status`
- 检查防火墙：
  ```bash
  # Ubuntu
  ufw status
  ufw allow 80/tcp
  ufw allow 443/tcp
  
  # CentOS/Rocky
  firewall-cmd --list-all
  firewall-cmd --permanent --add-port=80/tcp
  firewall-cmd --permanent --add-port=443/tcp
  firewall-cmd --reload
  ```
- 检查安全组：阿里云控制台 → 安全组 → 规则 → 入方向
