## 1. 个人信息

**文件位置**: `content/profile.md`

修改个人信息、技能、社交链接等。

### 字段说明

```yaml
---
name: "你的名字"                    # 显示名称
title: "你的职位/头衔"              # 副标题
avatar: "/images/avatar/avatar.png" # 头像路径
bio: |                              # 个人简介（支持多行）
  这里写个人简介...
  可以换行

location: "城市, 国家"              # 所在地
email: "your@email.com"             # 邮箱
website: "https://yoursite.com"     # 个人网站

social:                             # 社交链接（可选，不需要的可以删除）
  github: "https://github.com/xxx"
  twitter: "https://twitter.com/xxx"
  linkedin: "https://linkedin.com/in/xxx"
  bilibili: "https://space.bilibili.com/xxx"

skills:                             # 技能展示
  - category: "技能分类名"
    items:
      - name: "技能名称"
        level: 95                   # 熟练度 0-100

stats:                              # 统计数据
  yearsExperience: 5                # 从业年限
  projectsCompleted: 30             # 完成项目数
  linesOfCode: "500K+"              # 代码行数
---
```

### 修改示例

把技能熟练度从 95 改为 90：

```yaml
skills:
  - category: "前端开发"
    items:
      - name: "React/Next.js"
        level: 90    # 修改这里
```

添加新的社交链接：

```yaml
social:
  github: "https://github.com/xxx"
  twitter: "https://twitter.com/xxx"
  zhihu: "https://www.zhihu.com/people/xxx"  # 新增知乎
```

## 2. 项目内容

**文件位置**: `content/projects/` 目录

每个项目一个 `.md` 文件。

### 创建新项目

在 `content/projects/` 创建新文件，如 `my-new-project.md`：

```yaml
---
title: "项目标题"
slug: "my-new-project"                    # URL路径标识（英文，无空格）
description: "项目简短描述（显示在卡片上）"
date: "2024-05-20"                        # 发布日期
status: "completed"                       # 状态: completed | in-progress | archived
featured: true                            # 是否为精选项目（显示在首页）
thumbnail: "/images/projects/xxx.png"     # 封面图片
gif: "/videos/xxx.mp4"                    # 演示视频/GIF（可选，优先级高于thumbnail）

techStack:                                # 技术栈标签
  - "React"
  - "TypeScript"
  - "Node.js"

links:                                    # 项目链接（可选）
  demo: "https://demo.example.com"
  github: "https://github.com/xxx/project"

highlights:                               # 项目亮点（显示在卡片上）
  - "亮点1"
  - "亮点2"
  - "亮点3"
---

## 项目背景

这里写项目背景介绍...

## 技术实现

### 技术点1

详细说明...

### 技术点2

详细说明...

## 项目成果

- 成果1
- 成果2
```

### 修改现有项目

找到对应文件直接编辑即可。

### 删除项目

删除 `content/projects/` 下对应的 `.md` 文件。

## 3. 博客文章

**文件位置**: `content/blog/` 目录

每篇文章一个 `.md` 文件。

### 创建新文章

在 `content/blog/` 创建新文件，如 `my-new-article.md`：

```yaml
---
title: "文章标题"
slug: "my-new-article"                    # URL路径标识（英文，无空格）
description: "文章摘要（显示在列表和SEO）"
date: "2024-05-20"                        # 发布日期
tags:                                     # 标签
  - "标签1"
  - "标签2"
category: "分类名"                        # 分类
readingTime: 10                           # 预计阅读时间（分钟）
pinned: false                             # 是否置顶（可选，默认false）
---

## 引言

文章开头...

## 正文标题1

正文内容...

### 小标题

更多内容...

```python
# 代码示例
def hello():
    print("Hello World")
```

## 总结

总结内容...

### 表格

项目文档支持表格渲染：

```markdown
| 功能 | 状态 | 耗时 |
| --- | --- | --- |
| QA 工作流 | ✅ | ~20s |
| Draft 工作流 | ✅ | ~170s |
```

### 文章功能

**置顶文章**：设置 `pinned: true`，文章会显示在列表顶部并带置顶标识。

**代码高亮**：使用三个反引号 + 语言名：
````markdown
```typescript
const hello = "world";
```
````

支持的语言：`javascript`, `typescript`, `python`, `java`, `go`, `rust`, `html`, `css`, `bash`, `json`, `sql` 等。

**标题层级**：
- `##` - 二级标题（显示在目录）
- `###` - 三级标题（显示在目录）
- `####` - 四级标题（不显示在目录）

### 修改/删除文章

与项目相同，直接编辑或删除对应文件。

## 4. 图片资源

**文件位置**: `public/images/` 目录

### 目录结构

```
public/images/
├── avatar/           # 头像
│   └── avatar.png
├── projects/         # 项目图片
│   ├── project1.png
│   └── project2.png
├── blog/             # 博客配图
│   └── article1.png
└── og-image.png      # 社交分享预览图

public/videos/        # 项目演示视频
├── demo1.mp4
└── demo2.mp4
```

### 使用图片

在 Markdown 中引用：
```markdown
![图片描述](/images/projects/xxx.png)
```

在代码中引用：
```tsx
<img src="/images/avatar/avatar.png" alt="头像" />
```

### 图片建议

- **头像**: 建议 400x400px，PNG 或 JPG
- **项目封面**: 建议 1200x630px（与 OG 图片一致）
- **项目截图**: 建议 1920x1080px 或等比例
- **格式**: PNG（截图/UI）或 JPG（照片），可使用 WebP 压缩

### 视频建议

- **演示视频**: 建议 10-30秒，MP4 格式
- **文件大小**: 建议控制在 2MB 以内
- **分辨率**: 1280x720px 或 1920x1080px
- **用法**: 在项目文档中添加 `gif: "/videos/xxx.mp4"` 字段

## 5. 主题样式

**文件位置**: `app/globals.css`

### 主题色变量

```css
:root {
  /* 背景色 */
  --forest-bg: #1a2a3a;           /* 主背景 */
  --forest-bg-secondary: #243447; /* 次级背景 */

  /* 文字色 */
  --forest-text: #e8e8e8;         /* 主文字 */
  --forest-text-muted: #8899aa;   /* 次要文字 */

  /* 强调色 */
  --forest-accent: #f5a623;       /* 金色强调 */
  --forest-accent-hover: #d4921f; /* 悬停色 */

  /* 边框色 */
  --forest-border: rgba(255, 255, 255, 0.1);
}
```

### 修改主题色

把金色强调色改为蓝色：
```css
--forest-accent: #3b82f6;         /* 蓝色 */
--forest-accent-hover: #2563eb;   /* 悬停蓝色 */
```

把背景改为深黑色：
```css
--forest-bg: #0a0a0a;
--forest-bg-secondary: #1a1a1a;
```

## 6. 页面SEO

### 全局SEO

**文件位置**: `app/layout.tsx`

```tsx
export const metadata: Metadata = {
  title: {
    default: "PORTAL | 个人作品集",
    template: "%s | PORTAL",
  },
  description: "探索我的编程世界 - 项目作品与技术博客",
  keywords: ["程序员", "作品集", "技术博客"],
  openGraph: {
    url: "https://yoursite.com",      // 改成你的域名
    images: ["/og-image.png"],
  },
};
```

### 页面独立SEO

在每个页面的 `page.tsx` 中添加：

```tsx
export const metadata: Metadata = {
  title: "页面标题",
  description: "页面描述",
};
```

### 社交分享图

替换 `public/og-image.png`，建议尺寸 1200x630px。

## 常用操作

### 添加新项目
1. 复制 `content/projects/pixel-game-engine.md` 为模板
2. 重命名并修改内容
3. 添加封面图片到 `public/images/projects/`

### 添加新博客
1. 复制 `content/blog/learning-journey.md` 为模板
2. 重命名并修改内容
3. 可选：添加配图到 `public/images/blog/`

### 更新个人信息
直接编辑 `content/profile.md`

### 修改主题颜色
编辑 `app/globals.css` 中的 CSS 变量

### 部署更新
```bash
npm run build   # 构建
npm start       # 本地预览生产版本
# 或直接推送到 Vercel 自动部署
```
